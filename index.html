Deployment Guide

One-Click Installer Detailed Guide

Important Installation Notes

  • The API requires PHP with mod_rewrite enabled
  • Make sure your server has write permissions for the data directory
  • Always use the test.php script to verify your installation
  • If uploading to csvscrub.com/api, make sure all files are in that directory
  • Ensure the .htaccess file has proper RewriteBase setting (e.g., RewriteBase /api/)
  • The .htaccess file is critical but may be hidden in file managers

Deploy to SiteGround

  1. Login to your SiteGround account
  2. Go to Site Tools > Site > FTP
  3. Create an FTP account if you don't have one
  4. Download the installer from the "One-Click Installer" button above
  5. Upload all files to your desired directory (e.g., /public_html/api/)
  6. Visit your-domain.com/api/test.php to verify installation

Deploy to Netlify

  1. Push this code to a GitHub repository
  2. Sign up for Netlify
  3. Click "Add new site" → "Import an existing project"
  4. Connect to your GitHub repository
  5. Use these build settings:
    • Build command: npm run build
    • Publish directory: dist
  6. Click "Deploy site"
Go to Netlify

Deploy to Vercel

  1. Push this code to a GitHub repository
  2. Sign up for Vercel
  3. Click "Add New" → "Project"
  4. Import your GitHub repository
  5. Vercel will automatically detect it's a Vite project
  6. Click "Deploy"
Go to Vercel

Manual Deployment

  1. Download all files using the installer
  2. Upload files to your web server:
    • HTML, CSS, and JS files to your web root
    • PHP files to your API directory
  3. Ensure proper file permissions: 
    chmod 755 /path/to/api/data
chmod 644 /path/to/api/*.php
  4. Update config.php with your domain and API key
  5. Test the installation by visiting your-domain.com/api/test.php

Apache Configuration Note

If you're experiencing 404 errors, your Apache server might need additional configuration:

  1. Check if mod_rewrite is enabled: sudo a2enmod rewrite
  2. Edit your VirtualHost configuration file (often in /etc/apache2/sites-available/) to include:
    3. <Directory /path/to/your/api>
    Options Indexes FollowSymLinks
    AllowOverride All
    Require all granted
</Directory>
  3. Restart Apache: sudo systemctl restart apache2

API Usage Instructions

                
                

                  

                    
Endpoint URL

                    

                      https://your-domain.com/api/data
                    

                  

                  
                  

                    
Authentication

                    

                      Include your API key in the request headers:
                    

                    

                      X-API-Key: your-api-key-here
                    

                  

                  
                  

                    
Example Request

                    


fetch('https://your-domain.com/api/data', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-API-Key': 'your-api-key-here'
  },
  body: JSON.stringify({
    "source": "sales-system",
    "data": [
      {"id": 1, "value": 123.45, "date": "2023-07-15"},
      {"id": 2, "value": 67.89, "date": "2023-07-15"}
    ]
  })
});
